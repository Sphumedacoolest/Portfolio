how it works

Algorithms

A* (A-Star) OPTIMAL
Uses a heuristic (estimated distance to end) to prioritise promising paths. Guarantees the shortest route while exploring far fewer nodes than Dijkstra's.
Dijkstra's OPTIMAL
Explores all neighbours by cost, expanding outward like a flood. Always finds the shortest path but visits more nodes than A*.
BFS OPTIMAL
Breadth-First Search explores layer by layer. Optimal for unweighted grids. Uses a queue — first in, first out.
DFS NOT OPTIMAL
Depth-First Search dives as deep as possible before backtracking. Fast but does not guarantee the shortest path. Uses a stack.

Core Logic

astar.js — heuristic search JS
// Manhattan distance heuristic function heuristic(a, b) { return Math.abs(a.row - b.row) + Math.abs(a.col - b.col); } function aStar(grid, start, end) { const open = [start]; start.g = 0; start.f = heuristic(start, end); while (open.length) { // Pick node with lowest f score const cur = open.sort( (a, b) => a.f - b.f ).shift(); if (cur === end) return tracePath(end); for (const nb of getNeighbours(cur)) { const g = cur.g + 1; if (g < nb.g) { nb.parent = cur; nb.g = g; nb.f = g + heuristic(nb, end); if (!open.includes(nb)) open.push(nb); } } } return null; // no path found }
grid.js — path trace JS
// Walk back from end via parents function tracePath(endNode) { const path = []; let cur = endNode; while (cur) { path.unshift(cur); cur = cur.parent; } return path; }

Features
