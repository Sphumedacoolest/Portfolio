how it works

The Logic

WPM Calculation
WPM is calculated as (correct chars / 5) / minutes. Dividing by 5 is the industry standard — it normalises for word length so "cat" and "extraordinary" count fairly.
Live DOM Rendering
Every character is a separate <span> element. On each keystroke the app compares typed input to the target word and toggles .correct / .wrong classes in real time.
No Libraries
Everything — timer, rendering, WPM math, cursor tracking is pure vanilla JavaScript.
the code

Core Logic

typing.js — WPM formula JS
// Industry-standard WPM formula function calcWpm(correctChars, seconds) { const minutes = seconds / 60; // Divide by 5 = average word length return Math.round( (correctChars / 5) / minutes ); } // Accuracy = correct / total typed function calcAccuracy(correct, total) { if (total === 0) return 100; return Math.round((correct / total) * 100); }
typing.js — char render JS
// Render each char as its own span function renderWords(words) { display.innerHTML = words .map((word, wi) => ` <span class="word" id="w${wi}"> ${[...word].map((ch, ci) => `<span class="char pending" id="c${wi}-${ci}">${ch}</span>` ).join('')} </span>` ).join(''); }
what I built

Features

⌨️
Live Highlighting
Each character updates green or red in real time as you type — character by character, not word by word.
📊
WPM + Accuracy
Live WPM calculated using the industry-standard chars/5 formula. Accuracy tracks correct vs total keystrokes.
⏱️
4 Timer Modes
Choose 15, 30, 60, or 120 second rounds. Timer only starts on your first keystroke.
📝
3 Word Sets
Common English words, programming keywords, or famous quotes — different challenges for different goals.
🏁
Results Breakdown
End-of-round panel shows WPM, accuracy, error count, and raw WPM (unpenalised) side by side.
⚡
No Dependencies
Pure JavaScript : no frameworks, no npm, no build step. Just HTML, CSS, and JavaScript from.
